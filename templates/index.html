Our Mission

We set out with a goal to use Natural Language Processing (NLP) to find connections between various topics and a training set. In our case, we used Latent Symantic Indexing (LSA) to train a Python module called Gensim to understand it means for a word to be a "software engineering" term. Then, we took in user-provided terms through a search feature, and used our model to identify the top 10 software engineering topics most related to that term.

Why

Software engineering is an enormous field that uses quite a bit of jargon. The goal of this application was to allow anyone to find the top similarities between any term that they searched, and the topics in our database of software engineering terms.

The How

We used a Python web crawler, combined with the Wikipedia API to compile an enormous list of Wikipedia pages related to software engineering. Then, we stored these pages in a Mongo Database. After we were able to store these articles, we fed them to the Gensim Python module, which "tokenized" the data from the articles, turning the data into a series of vector relations, and creating an LSA model of the data. From there, we took user input through the search bar, and used the LSA model to provide the user with the top 10 software engineering topics related to their search query.

Technology Used:
  • Gensim (Python Module) - Easy topic modelling engine
  • Wikipedia API (Python Module) - Retrive and compile article information
  • Web Crawler (Homegrown) - Crawl Wikipedia to build initial list of software engineering terms for our training set
  • MongoDB (Database) - Stored our training set
  • Flask (Python Module) - Lightweight MVC module for Python
